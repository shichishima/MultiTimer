
これはmacOS＆iOS用のGUIアプリ「マルチタイマー」の仕様です。

# 概要
- 画面上で同時に複数のカウントダウンタイマーを表示する
- タイマーは縦横の表形式で表示する
- それぞれのタイマーは単独動作のものと、同じタイマーが2か所のセルで表示されるものがある
- このアプリの複数起動は考慮しない
- 個人用途の簡易アプリであるため、エッジケースにはあまり拘らず、アプリ再起動等の運用でカバーできる範囲はそれで良いものとする。

# 表形式でのタイマー表示
- 横方向に最小1〜最大5のカラムがあり、それぞれが「利用者」をあらわす
	- ここでは「利用者A」「利用者B」「利用者C」「利用者D」「利用者E」とする
	- 左端に見出しカラムはない
	- なお設定画面で登録する利用者は6人だが、「表示する」が最大5人のため最大5カラムである。
- 縦方向には見出し行を含め5行ある
	- 第1行は見出し行として利用者名を表示する
	- 2行目〜5行目は「タイマー行」である。利用者ごとに最大4つのタイマーとなる
		- うち一つは非連携タイマー(必ずスロット1つ存在する)、最大3つが連携タイマー(オプショナルでスロット0個〜3個)
	- 各カラムについて、タイマー行は残り時間の少ない順に表示する
		- 新規にタイマーを開始したり、途中でタイマーを停止(削除)した場合に表示順序が変化する
		- 残り時間が同じ場合の表示順は任意
- タイマー行のセルは次の2種類(3状態)
	- タイマーセル
		- タイマー種類は「利用者自身が使うタイマー」「別の利用者と連携利用するタイマー」の2種類
		- それぞれ、状態は「動作中(カウントダウン中)」「停止中(カウントゼロ)」の2種類
	- 空セル
		- タイマーが設定されていないセル(連携数が3未満のもの)は空セルとする。
		- それぞれのカラムでは、タイマーセルを上、空セルを下に並べる
- アプリ起動時および設定変更時に全てのタイマーセルが作成されている
	- 利用者名、連携状態も反映させる
	- 初期状態では「0:00」を表示する。
- 表示ウィンドウのサイズは可変とする
	- ウィンドウサイズに応じてフォントサイズを調整する
	- 最大化機能、全画面機能は不要



# タイマーセル
- カウントダウンタイマーである
- セルの表示項目は以下の通り
	- 色見出し
		- 色見出しは後述の「2つのタイマーの連係」を示す色
		- セルの左側に縦にライン上に見やすく配置する
	- 連携対象がいる場合には、連携先の利用者名
		- 非連携タイマーはその箇所を空欄にする
	- タイマー残り時間 (大きく)
	- 元のタイマー時間 (残り時間の下に小さく)
		- 元のタイマー時間が未設定の場合(アプリ起動時)は表示しない
- タイマー動作中のセルは
	- クリックするとセル全体が半透明のオーバレイに覆われ、上に「×」ボタンが表示される
		- ×ボタン表示中もタイマーのカウントダウンは動作し続ける
		- ×ボタンを押すとそのカラムのタイマーは「0:00」にリセットされ、動作停止する
			- これにより同じカラムの各セルの表示順序は変化する
			- 連携先のタイマーセルも同様に0:00にリセットされ動作停止する
			- 「元のタイマー時間」は維持する
		- ×ボタン以外の場所を押すと×ボタンが消えオーバレイが元に戻る
	- タイマーが0:00になったら
		- システムで持っている既存の「ポン」とサウンドを鳴らす
		- 0:00の文字を赤文字にする
- タイマー動作停止中のセル(0:00表示で止まっているセル)は
	- クリックするとオーバレイで2つのボタンを表示する
		- 丸矢印：元のタイマー時間がセットされている場合に表示する。ボタン押下で同じ時間のタイマーを再度開始する
		- ストップウォッチアイコン(新規タイマー)：クリックすることで新規のカウントダウンタイマーを設定し、即座にカウントダウン開始する。
	- タイマーは12時間、8時間、6時間、4時間、3時間、2時間、90分、1時間、45分、30分、20分、10分、5分、3分、1分、からポップオーバー型で選択する
		- カスタム時間は設定できなくて良い
		- デフォルト選択値はなし
		- ポップオーバー外をクリックすることでキャンセル
	- 連携状況は変わらない。設定済みのものを踏襲する
	- 残り時間順での表示であるから、カラム内の表示順は他の動作中タイマーより上。(見出しセルの下)
	- 「すでにカウントダウンが終了したタイマー」も同様の表示とする。
		- アプリ停止中にカウントダウンが満了した場合。この場合はアプリ起動時に通知音を鳴らす。(複数のカウントダウン満了タイマーがあっても一度だけ鳴らす)
- 設定されていないセルは「空セル」
	- 何もアクションがない。完全な空白セル



# 設定画面
- command＋, で設定画面が開く
- 設定画面上部に「利用者」「連携」の切り替えタブをおく。
- 「利用者」タブ：利用者名称の設定、カラム順序の設定
	- 6行の設定項目があらかじめ表示されている
	- 1行ごとに1人の利用者を登録できる
	- それぞれの利用者の画面上の表示名称を設定できる
	- それぞれの利用者について画面にカラムを割り当てるかどうか設定できる
		- オンにできるのは最小1人〜最大5人
		- 初期状態で最初の1人がオンになっている(最小1を担保)
		- 5人オンになったら残りの1人の設定項目はグレーアウトする(最大5を担保)
		- 非表示にした場合にはメイン画面の表示カラムが減る
			- 非連携タイマーは「0:00」にリセットされ停止する
			- 連携先の連携タイマーは表示停止の影響を受けない。連携先では表示され、動作は継続し、連携先で操作可能
	- 行をドラッグすることでカラムの表示順序を変更できる
- 「連携」タブ：画面表示する設定の利用者について連携状態(ほかの利用者)の設定
	- 3行 x 6カラムの表形式になっている
	- 各カラムが利用者。見出し行に利用者名を表示する
	- 利用者ごとに、連携相手を3人までセットできる
		- 各利用者ごとに「連携枠」が3つある。自分以外の利用者と連携できる
		- 「利用者」で設定した利用者から、プルダウンで選択する
			- 自分自身は選択できない
			- すでに連携枠を3つ使っている利用者も選択できない
		- Xボタンで連携設定をクリアできる
		- それぞれの連携相手ごとに「色見出し」を設定できる
			- 初期状態では全て同一の固定色とする。(色は任意)
			- カラースポイト機能(macOS標準で良い)でデスクトップ上からピックアップする
		- 連携は常に双方向
			- 一つ設定すると自動的に逆方向の連携がセットされる
			- 色見出しは同一
		- 連携設定を変更した場合には連携タイマーは0:00にリセットする

# 2つのタイマーの連係動作
- 設定画面の利用者連携登録により、表形式のタイマーセルは同じタイマーを表示する場合がある。
	- 例えば、利用者Aと利用者Bが連携している場合、利用者Aのカラムにある利用者Bのタイマーと、利用者Bのカラムにある利用者Aのタイマーは同じものとなる
	- どちらかを止めれば両方とまる。どちらかのタイマー時間を変更すれば両方変わる。
	- 同じタイマーが2箇所に表示されている状態。(タイマーセルの見出し色は同じだが、連携先利用者名は異なる)
	- 連携タイマーは利用者A・利用者Bどちらからでも操作できる

# データ永続化
- アプリ終了後も設定内容は保存する
- アプリ停止時もカウントダウンが継続しているように振る舞う
	- 残り時間を保持するのではなく、0:00となるべき未来時刻を保持することで長時間のアプリ停止に対応する
	- このためウィンドウを閉じたバックグラウンドで動作する必要はなく、アプリ終了して良い
- 永続データはiCloud経由でmacOS〜iOSどちらでも利用可能とする。
	- 変更の競合に対しては、後の更新を有効とする
	- データ変更を即時に反映する
	- NW未接続等、エラーの場合にはその旨表示する。対処不要
- データ暗号化不要


# iOSアプリ
- iOS26で動けばよい (iOS26.0以降対応)
- 縦画面の場合
	- 利用者を表示している部分を左右スワイプで左右スクロールする
- 横画面の場合
	- 全体を表示。フォントサイズを適宜調整し、表示すべき行・列をすべて表示する
- 設定画面不要
	- 起動時に設定データが読み込めない、あるいはデータがない場合はその旨画面表示をして動作停止。再起動時にフォローする
	- mac版で設定が完了していることを前提とする
- バックグラウンド動作中にカウントダウンが満了した場合には通知(ローカル通知)する
