## A. コマンドラインで指示した変更

### A-1. ⚠️ iCloud → ローカルYAML に変更

**指示:** 「一旦icloud利用せずローカルにファイルでデータを保存して。yaml形式で」

**変更内容:**
- `TimerStore.swift` の iCloud (`NSUbiquitousKeyValueStore`) を削除
- `LocalStorage.swift` を新規作成（手書き YAML エンコーダー／デコーダー）
- 保存先: `~/Library/Application Support/naoki.MultiTimer/data.yaml`
- 書き込みエラーは `saveErrorMessage` プロパティ経由でメイン画面上部に赤帯表示

---

### A-2. ⚠️ カスタム時間ピッカーの追加

**指示:** 「タイマー時間のプルダウンに『カスタム』を追加して。任意の時間をキーボードから設定できるようにして」

**変更内容（`TimerCellView.swift` の `DurationPickerView`）:**
- プリセット一覧の末尾に「カスタム」行を追加
- タップで時間・分・秒の入力フォームがアニメーション展開
- リアルタイムプレビュー表示（`formatTime()` で整形）
- Enter キー（`.keyboardShortcut(.return, modifiers: [])`）で「開始」ボタン実行

> **仕様:** 「カスタム時間は設定できなくて良い」→ 意図的に追加

---

### A-3. ⚠️ タイマー残り時間の表示アライン（2段階で変更）

**指示①:** 「タイマーセルについてタイマー残り時間の表示を右揃えにして」
**指示②:** 「タイマー残り時間の『表示枠』はセルに対してセンタリングする。ただし、タイマーの最大値を考慮して『表示枠』のサイズと位置を決定する。実際のタイマー残り時間は、表示枠の中で右揃えにする。」

**変更内容（`TimerCellView.swift`）:**
```swift
ZStack(alignment: .trailing) {
    Text("12:00:00")          // 最大幅の基準（非表示）
        .font(timeFont)
        .hidden()
    Text(formatTime(slot?.remainingSeconds ?? 0))
        .font(timeFont)
        .foregroundStyle(isCompleted ? .red : .primary)
        .lineLimit(1)
        .minimumScaleFactor(0.6)
}
```
- 非表示の「12:00:00」テキストで ZStack の幅を最大値に固定
- その ZStack を VStack → `frame(maxWidth: .infinity)` でセル内センタリング
- 実際の時間テキストは ZStack 内で右揃え（`.trailing`）

---

### A-4. ⚠️ 利用者名の編集を「表示ON時のみ」に制限

**指示:** 「設定画面で『表示ON』の時にテキスト編集できるようにして」

**変更内容（`SettingsView.swift`）:**
```swift
TextField("名前", text: $editedUsers[i].name)
    .disabled(!editedUsers[i].isVisible)
```
- `isVisible = false` の利用者の TextField を disable
- 仕様には表示状態による制限の記載なし → 意図的追加

---

### A-5. ⚠️ デフォルト利用者名の変更

**指示:** ユーザーが `Models.swift` を直接編集

**変更内容（`Models.swift`）:**

### A-6. ⚠️ アプリアイコンの設定

**指示:** 「マルチタイマーに相応しいアイコン画像をセットして」

**変更内容:**
- `/tmp/generate_icon.swift`（AppKit + CGContext による描画スクリプト）を作成・実行

## B. 開発中に発生・修正したバグ

### B-1. 設定画面の TextField フォーカス不可

**症状:** 「利用者の設定画面で利用者名をクリックしてもダブルクリックしてもテキスト入力にならず変更できない」

**原因:**
- `ForEach($editedUsers) { $user in }` + `List` の組み合わせで macOS がクリックイベントを List の行選択として消費し、TextField まで届かなかった
- `.listStyle(.inset(alternatesRowBackgrounds: true))` の影響も疑われた

**修正内容（`SettingsView.swift`）:**
```swift
// 変更前
ForEach($editedUsers) { $user in
    Toggle("", isOn: $user.isVisible)
    TextField("名前", text: $user.name)
}
// 変更後
ForEach(editedUsers.indices, id: \.self) { i in
    Toggle("", isOn: $editedUsers[i].isVisible)
    TextField("名前", text: $editedUsers[i].name)
}
```
- インデックスベースの ForEach + `.listStyle(.inset)` に変更することで解消

---

### B-2. タイマーカウントダウンが画面に反映されない

**症状:** 「内部的なタイマーのカウントダウンは進行しているようだが、画面表示に反映されない」

**原因:**
- `TimerSlot.remainingSeconds` は `Date()` から計算されるが、`store.data`（AppData 構造体）は毎秒変化しないため `@Observable` の依存追跡がトリガーされず SwiftUI が再描画をスキップしていた

**修正内容:**

`TimerStore.swift` に `tick` プロパティを追加:
```swift
var tick: Int = 0  // 毎秒インクリメントして再描画トリガーに使用

private func checkForNewlyExpired() {
    tick += 1  // 追加
    // ... 既存の満了チェック処理
}
```

`ContentView.swift` と `TimerCellView.swift` に依存を追加:
```swift
var body: some View {
    let _ = store.tick  // tick を読むことで毎秒再描画される
    return ZStack { ... }
}
```

## C. 実装から読み解いた仕様との差分（未対応）

### C-1. ❌ iOSターゲットが未追加

**仕様:** iOS26で動けばよい（iOS26.0以降対応）

**実装:**
- Xcodeプロジェクトは **macOS単一ターゲット** のまま
- Swift コード内に `#if os(iOS)` 分岐は存在するが、iOSビルドは不可
- Xcodeで手動でiOSターゲットを追加する必要がある

---

### C-2. ❌ iOS起動時の設定なしエラー表示

**仕様:** 起動時に設定データが読み込めない、あるいはデータがない場合はその旨画面表示をして動作停止。

**実装:**
- データがない場合は `AppData()` のデフォルト値（なー 1人のみ有効）にフォールバックして起動
- エラー表示・動作停止の処理は未実装

---

### C-3. ❌ ローカル通知（コード実装済み・動作未確認）

**仕様:** バックグラウンド動作中にカウントダウンが満了した場合には通知（ローカル通知）する。

**実装:**
- `TimerStore.swift` 内に `scheduleNotification` / `cancelNotification` を `#if os(iOS)` で実装済み
- iOSターゲットが存在しないためビルド・動作確認不可

---

### C-4. ❌ 利用者の「最小1人ON」担保が不完全

**仕様:** オンにできるのは最小1人〜最大5人

**実装:**
- 最大5人の上限制御: ✅ （5人ONで残りのトグルをdisable）
- 最小1人の下限制御: ❌ 現在ONの利用者が1人だけの場合でも、そのトグルをOFFにできてしまう（全員OFFが可能）

**修正案:**
```swift
.disabled(
    (!editedUsers[i].isVisible && visibleCount >= 5) ||
    (editedUsers[i].isVisible && visibleCount <= 1)   // ← この行が不足
)
```

---

### C-5. ❌ 連携タブに非表示ユーザーも表示される

**仕様:** 「連携」タブ：**画面表示する設定の利用者**について連携状態（ほかの利用者）の設定

**実装:**
- 連携タブの列は**全6ユーザー**を常に表示
- `isVisible = false` の利用者も列として表示される

**修正案:** `LinksSettingsView` の列表示を `store.data.visibleUsers` でフィルタする（連携相手の候補Menu側は引き続き全6人を対象にして良い）。

---

### C-6. ❌ フルスクリーン・最大化ボタンの無効化

**仕様:** 最大化機能、全画面機能は不要

**実装:**
- macOS の `WindowGroup` 標準のままのため、タイトルバーにフルスクリーンボタン（緑の丸）が表示される

**修正案:**
```swift
WindowGroup {
    ContentView().environment(store)
}
.windowResizability(.contentMinSize)
// もしくは NSWindow 経由で styleMask から .fullScreen を除く
```

## D. まとめ: 優先度別の残タスク

| 優先度 | 項目 |
|--------|------|
| 高 | iOSターゲットの追加（Xcodeで手動作業が必要） |
| 高 | iCloud永続化への差し戻し（またはiOSとのファイル共有方式の確立） |
| 中 | 利用者最小1人ONの担保（Toggle の disable条件に1行追加） |
| 中 | 連携タブの列を表示中ユーザーのみに絞る |
| 低 | フルスクリーン・最大化ボタンの非表示 |
| 低 | iOS起動時の設定なしエラー画面 |
