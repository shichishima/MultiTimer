# 連携タイマーセルへのチェックボックス設置
- 各利用者の備忘のため、各タイマーセルの右下にチェックボックスを設置する
	- 空セルには不要
	- 非連携タイマーには不要
	- 連携タイマーは動作中/停止中どちらも必要
- 連携タイマーであってもチェック状態は「セルごと個別に」設定可能とする
	- チェックボックスのON/OFF状態は連携しない。個別に操作可能とする
- ON/OFF状態は保存する
	- タイマーデータと同じファイルに保存し、アプリ再起動で状態を復元する

# 永続データの保存と読み出し
- 永続データの共有にiClaudは使わない
	- 既存のiCloud関連コードを削除する
	- 現状のまま、永続データはローカルファイルとして保存する
- ただし、永続データファイルの保存位置をユーザが指定できるようにする
	- 設定画面に3番目のタブ「共有」を追加し、そこでデータ保存先を指定できるようにする
	- データファイル名は「MultiTimer.yml」とする
	- フォルダのみを指定する
	- macOS、iOSともにOS標準のファイルピッカーを使用する
		- iOS向けの実装は現時点では行わない。設定画面の持ち方を含め別途指示する
- 初期状態で保存フォルダが未設定の場合
	- メイン画面に「設定画面から共有データを指定してください」と表示する
	- 設定画面を開いたら「共有」タブを表示する
	- 設定画面で「利用者」「連携」タブはグレーアウトして押せないようにする
- 指定された保存フォルダにデータファイルが存在しない場合
	- まだ設定データがない場合(初期状態で初めて保存フォルダを指定して、そこにデータがなかった場合)は、新たにデータファイルを作成し保存する
	- 設定データがある場合(すでに利用中の状態で保存先を変更した場合)は、現状の設定内容を新たにデータファイル化して保存する
- GUI操作にて状態が変化したらその時点でファイルを更新する
- アプリ外でデータファイルが更新されることを想定する
	- データ項目の欠損は無視する
- 見出し行のダブルクリックでデータファイルを読み込み、表示状態を更新する
	- 読み込み中は画面全体をグレーアウトし中央に読み込み中表示を行う
- 背景情報
	- アプリ間の永続データ共有は、本アプリの管理外にて行う。
	- ユーザがファイル保存先としてGoogleDriveフォルダを指定することを想定する
	- GoogleDrive経由で(アプリ外で)データが更新される


# マウスオーバー時の挙動
- 連携タイマーセルの上にマウスカーソルがある場合、対応する他方の連携セルが視覚的にわかるようにする
	- タイマーセルクリック時と同様の半透明オーバレイ表示とする
