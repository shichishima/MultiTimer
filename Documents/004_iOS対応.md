
MultiTimerアプリについて、以下の方針でiOS(iOS26以降)に対応する。

# 0. 画面構成概要
- 縦画面専用とする
	- デバイスが横画面の場合でも縦に表示する
	- Info.plistで縦画面制限と制御する
- 画面最下部をタブエリアとし、「タイマー」「共有」ボタンをおく
	- 「タイマー」ボタン押下でメイン表示領域に表形式のタイマーを表示する
	- 「共有」ボタン押下でメイン表示領域に共有フォルダの設定画面を表示する
	- iOS版タブエリアはiOS用として別ファイルに切り分け、TabViewで実装する
- タブエリア以外をメイン表示領域とする
- タイマー表示の際のメイン表示領域では
	- 各セルは横200pt x 縦80ptを初期サイズとし、ピンチイン/ピンチアウトで拡大縮小できるものとする。(x0.5〜x1.2)
	- スクリーンを左右にスワイプすることで左右スクロールする
	- ピンチ操作とスワイプ操作の同時操作は共存させ、制御しない
	- 見出しセルをダブルタップすることでデータ再読み込み・タイマー満了チェックを行う

# 1. 共有フォルダ設定画面
- タブエリアの「共有」ボタンを押すことでメイン表示領域に共有フォルダ選択画面を表示する
	- タイマー表示と差し替える
	- 「閉じる」ボタンは設置しない。 (「タイマー」タブボタンを用いる)
- iOS標準のファイルピッカーを用いる
	- .fileImporterでallowedContentTypes: .folder を指定する
- 共有フォルダが設定されたら、setDataFolder()にてブックマーク保存する
- なお、利用者設定および連携設定はmacOS側で実施する前提とし、iOS版では設定画面を用意しない。アプリ内では案内も不要
  
  
# 2. TimerCellView のホバーへの対応
- iOS版では、マイスホバーの代わりにタイマーセルのタップ操作にてオーバレイ表示をトグルする
	- 連携するセルがある場合は同時にオーバレイする
		- 連携先セルでの表示内容はmacOS版同様、オーバレイだけでボタンなし
	- すでに別のセルがオーバレイ表示されている場合には、それ(連携を考慮すると2つありうる)のオーバレイを解除する
- オーバレイ状態はContentView側で管理し、排他制御する
- セル以外(空白エリア)をタップした場合はすべてのオーバレイを解除する
	- 空白セル(タイマー未割り当て)に.contentShape(Rectangle())と.onTapGestureを付与し、タップ時にactiveCell=nilとする
- macOS版についてもオーバレイ制御を統一的に扱うため、状態管理をContentViewに移管する
	- mac版でのホバー動作は、.onHoverの中でonHoverChanged(true)を呼び出すことで統一を図る
	- showDurationPickerはセルローカルのままとする。
	- タイマー開始時(DurationPickerViewのonSelect完了時)にはonHoverChanged(false)を呼び出してオーバレイを解除する
	- onHoverChangedの名称は維持し、iOS版ではタップ時にonHoverChanged(true)を呼ぶ
	- ホバーleaveに相当するonHoverChanged(false)はiOSでは呼ばない。(ContentViewがactiveCell切り替えで管理するため)

# 3. LocalStorage — iOSのブックマーク保存対応
- iOS用のコードブロックを以下の2箇所に追加する
	- LocalStorage.swift: saveBookmark()・resolveBookmark()のiOS版 (options:[]、staleチェックあり)
	- TimerStore.setDataFolder()： saveBookmark(url:)とstartAccessingSecurityScopedResource()の呼び出し
- アプリ再起動のたびにフォルダを再選択する必要をなくす
- iOS版では options:[] を用いる
- フォルダ選択後・ブックマーク解決後、ファイルI/O前に startAccessingSecurityScopedResource()を呼び出す
- startAccessing/stopAccessingの呼び出しタイミングは 5. のライフサイクル対応に従う


  
# 4. TimerStore.init() — iOSの初期化対応
- iOS版においても、共有フォルダ設定後にデータファイルを読み込む
- アプリ再起動後にも(共有フォルダが設定されている場合には)resolveBookmark()でURLを解決し、データファイルを読み込む
- ブックマーク解決後、startAccessingSecurityScopedResource()を呼んでからloadFromDisk()を実行する


# 5. バックグラウンドライフサイクル対応
- 以下の2つの通知をTimerStore.init()内で購読する
	- didBecomeActiveNotification(フォアグラウンド復帰時)
		- startAccessingSecurityScopedResource()を呼んでからreloadFromFile()を実行する
		- reloadFromFile()の完了時(Task内)にタイマー満了チェックを行う
	- didEnterBackgroundNotification(バックグラウンド移行時)
		- stopAccessingSecurityScopedResource()を呼ぶ

  
# 6. iOS向けの「データ未設定時」フロー
- iOS版でデータ未設定の場合には、タブボタン「共有」が押された状態の画面とする
	- この時「タイマー」ボタン押下時には、メイン表示領域にはタイマーを表示せず、「共有フォルダーを設定してください」とだけ表示する。誘導ボタンは設置しない
	- 設定完了後、自動的に「タイマー」タブを表示する
	- selectedTabはiOSルートViewの@stateとしてもち、store.dataFolderURLの変化を.onChangeで監視して自動切り替えする

  
# 7. データ共有方法
- iOS版では、別途GoogleDriveアプリがインストールされ、ファイルアプリと連携されていることを前提とする
	- アプリ側では「フォルダを指定してください」とするだけ
- iOS版のresolveBookmark()でも、isStaleがtrueの場合は再保存する
